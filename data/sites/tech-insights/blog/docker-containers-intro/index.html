Introduction to Docker Containers

Author: James Rodriguez Published: January 21, 2025

Docker has revolutionized how we build, ship, and run applications. Containerization provides consistency across development, testing, and production environments while improving resource utilization. This guide introduces Docker and shows you how to get started with containers.

What Are Containers?

Containers package application code together with all dependencies, libraries, and configuration files needed to run. Unlike virtual machines, containers share the host system's kernel, making them lightweight and fast to start.

Why Docker?

  • Consistency: "It works on my machine" becomes a thing of the past
  • Isolation: Applications run in isolated environments
  • Portability: Run anywhere Docker is supported
  • Efficiency: Containers are lightweight compared to VMs
  • Scalability: Easy to scale applications horizontally

Core Concepts

Images

A Docker image is a read-only template containing application code, runtime, libraries, and dependencies. Images are built from Dockerfiles and serve as the blueprint for containers.

Containers

Containers are running instances of images. You can have multiple containers running from the same image, each isolated from the others.

Dockerfile

A Dockerfile is a text file containing instructions to build a Docker image.

# Example Dockerfile for a Node.js app
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .

EXPOSE 3000

CMD ["node", "server.js"]

Essential Docker Commands

# Pull an image from Docker Hub
docker pull nginx

# Run a container
docker run -d -p 80:80 nginx

# List running containers
docker ps

# Stop a container
docker stop container_id

# Remove a container
docker rm container_id

# Build an image from Dockerfile
docker build -t myapp:1.0 .

# View logs
docker logs container_id

Docker Compose

Docker Compose simplifies running multi-container applications. Define your application's services in a YAML file and start everything with a single command.

version: '3.8'
services:
  web:
    build: .
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgres://db:5432/mydb
    depends_on:
      - db

  db:
    image: postgres:15
    environment:
      - POSTGRES_PASSWORD=secret
    volumes:
      - postgres-data:/var/lib/postgresql/data

volumes:
  postgres-data:

Best Practices

  • Use official base images when possible
  • Keep images small by using multi-stage builds
  • Don't run containers as root
  • Use .dockerignore to exclude unnecessary files
  • One process per container (microservices approach)
  • Tag images with version numbers, not just 'latest'
  • Scan images for security vulnerabilities

Common Use Cases

  • Development environments that match production
  • Continuous integration and deployment pipelines
  • Microservices architectures
  • Testing applications in isolated environments
  • Running legacy applications on modern infrastructure

Docker is an essential tool in modern software development. Whether you're a developer looking to streamline your workflow or a DevOps engineer managing production systems, mastering Docker will make your life easier and your applications more reliable.
