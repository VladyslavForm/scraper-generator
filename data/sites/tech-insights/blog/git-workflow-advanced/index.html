Advanced Git Workflows

Author: Nathan Price Published: January 27, 2025

Git is more than just commit, push, and pull. Advanced Git workflows enable teams to collaborate effectively, maintain clean history, and manage complex projects with confidence. This guide explores sophisticated Git techniques for professional development teams.

Branching Strategies

Git Flow

Git Flow uses multiple long-lived branches for different purposes:

  • main: Production-ready code
  • develop: Integration branch for features
  • feature/*: Individual feature branches
  • release/*: Release preparation branches
  • hotfix/*: Emergency production fixes
# Start a feature
git flow feature start new-feature

# Finish a feature
git flow feature finish new-feature

# Start a release
git flow release start 1.0.0

GitHub Flow

Simpler workflow with main branch and feature branches:

# Create feature branch
git checkout -b feature/new-feature

# Make changes and commit
git add .
git commit -m "Add new feature"

# Push and create pull request
git push -u origin feature/new-feature

Interactive Rebase

Clean up commit history before merging:

# Rebase last 3 commits
git rebase -i HEAD~3

# In the editor, you can:
# pick - keep commit as is
# reword - change commit message
# squash - combine with previous commit
# fixup - like squash but discard message
# drop - remove commit

Cherry-picking

Apply specific commits to another branch:

# Apply commit to current branch
git cherry-pick abc123

# Cherry-pick multiple commits
git cherry-pick abc123 def456

# Cherry-pick without committing
git cherry-pick -n abc123

Stashing

Temporarily save changes without committing:

# Save current changes
git stash save "work in progress"

# List stashes
git stash list

# Apply most recent stash
git stash apply

# Apply and remove stash
git stash pop

# Apply specific stash
git stash apply stash@{2}

Bisect for Bug Hunting

Binary search to find which commit introduced a bug:

# Start bisect
git bisect start

# Mark current commit as bad
git bisect bad

# Mark last known good commit
git bisect good abc123

# Git will checkout middle commit
# Test and mark as good or bad
git bisect good  # or git bisect bad

# Reset when done
git bisect reset

Reflog: The Safety Net

Recover lost commits and branches:

# View reflog
git reflog

# Recover deleted branch
git checkout -b recovered-branch HEAD@{2}

# Undo recent commits
git reset --hard HEAD@{3}

Submodules and Subtrees

Submodules

# Add submodule
git submodule add https://github.com/user/repo.git path/to/submodule

# Clone repo with submodules
git clone --recurse-submodules https://github.com/user/main-repo.git

# Update submodules
git submodule update --remote

Subtrees

# Add subtree
git subtree add --prefix=lib/dependency https://github.com/user/dependency.git main

# Update subtree
git subtree pull --prefix=lib/dependency https://github.com/user/dependency.git main

Hooks for Automation

# .git/hooks/pre-commit
#!/bin/bash
npm run lint
npm test

# Make executable
chmod +x .git/hooks/pre-commit

Advanced Merge Strategies

# Recursive merge (default)
git merge feature-branch

# Ours strategy - keep our version
git merge -X ours feature-branch

# Theirs strategy - prefer their version
git merge -X theirs feature-branch

# Squash merge - combine all commits
git merge --squash feature-branch

Best Practices

  • Write clear, descriptive commit messages
  • Commit early and often (locally)
  • Keep commits atomic and focused
  • Use pull requests for code review
  • Protect main branch with branch policies
  • Use tags for releases
  • Document your workflow in CONTRIBUTING.md

Mastering advanced Git workflows makes you more effective and confident as a developer. These techniques help maintain clean history, facilitate collaboration, and provide safety nets when things go wrong. Practice these commands in a test repository to build muscle memory.
